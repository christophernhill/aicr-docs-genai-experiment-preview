    Running an LLM with vLLM¶

    Run large language model inference on AICR using vLLM with tensor parallelism across multiple GPUs.

    Goal¶

    Pull the vLLM container, write a batch inference script, and run a 70B-parameter model across 4 B200 GPUs using tensor parallelism (tensor_parallel_size=4).

    Prerequisites¶

    • An AICR account with a valid Slurm account (--account=ACCOUNT_NAME)
    • A Hugging Face account with access to the model (some models require accepting a license agreement on huggingface.co)
    • Sufficient GPU memory for your model (see Inference and Serving for sizing)

    Steps¶

    1. Pull the vLLM Container¶

    Containers are large (10+ GB). Pull on a compute node to avoid burdening login nodes:

    $ salloc --partition=cpu --cpus-per-task=4 --mem=32G --time=01:00:00 --account=ACCOUNT_NAME
$ mkdir -p /scratch/$USER/containers
$ apptainer pull /scratch/$USER/containers/vllm.sif docker://vllm/vllm-openai:latest
$ exit

    Tip

    Store .sif files on /scratch/$USER/containers/. They are easy to rebuild if purged.

    2. Configure the Hugging Face Cache¶

    Redirect the Hugging Face cache to scratch so model downloads do not fill your home quota:

    $ echo 'export HF_HOME=/scratch/$USER/.cache/huggingface' >> ~/.bashrc
$ source ~/.bashrc

    Set up your access token for gated models (e.g., Llama):

    $ mkdir -p /scratch/$USER/.cache/huggingface
$ echo "YOUR_HF_TOKEN" > /scratch/$USER/.cache/huggingface/token
$ chmod 600 /scratch/$USER/.cache/huggingface/token

    3. Write the Batch Inference Script¶

    Create /scratch/$USER/vllm-demo/batch_inference.py:

    """Batch inference with vLLM using tensor parallelism."""
import json
import os

from vllm import LLM, SamplingParams

output_dir = os.path.expandvars("/scratch/$USER/vllm-demo/results")
os.makedirs(output_dir, exist_ok=True)

llm = LLM(
    model="meta-llama/Llama-3.1-70B-Instruct",
    tensor_parallel_size=4,
    dtype="bfloat16",
)

sampling_params = SamplingParams(
    temperature=0.7,
    max_tokens=512,
    top_p=0.9,
)

prompts = [
    "Explain the significance of attention mechanisms in transformers.",
    "What are the main challenges in protein structure prediction?",
    "Summarize the key ideas behind reinforcement learning from human feedback.",
    "Describe three applications of large language models in scientific research.",
]

outputs = llm.generate(prompts, sampling_params)

results = []
for output in outputs:
    result = {
        "prompt": output.prompt,
        "generated_text": output.outputs[0].text,
    }
    results.append(result)
    print(f"Prompt: {output.prompt[:60]}...")
    print(f"Output: {output.outputs[0].text[:200]}")
    print("---")

output_path = os.path.join(output_dir, "results.json")
with open(output_path, "w") as f:
    json.dump(results, f, indent=2)

print(f"Results saved to {output_path}")

    4. Submit the Job¶

    Create vllm_job.sh:

    #!/bin/bash
#SBATCH --job-name=vllm-batch
#SBATCH --partition=b200-batch
#SBATCH --nodes=1
#SBATCH --gpus-per-node=4
#SBATCH --cpus-per-task=32
#SBATCH --mem=256G
#SBATCH --time=04:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge

export HF_HOME=/scratch/$USER/.cache/huggingface

apptainer exec --nv \
    --bind /scratch/$USER:/scratch/$USER \
    /scratch/$USER/containers/vllm.sif \
    python /scratch/$USER/vllm-demo/batch_inference.py

    $ sbatch vllm_job.sh
Submitted batch job 12345

    Troubleshooting¶

    Problem Cause Fix
    OutOfMemoryError during model loading Model too large for requested GPUs Increase tensor_parallel_size and --gpus-per-node (e.g., 8 GPUs for 405B models)
    Access to model is restricted Hugging Face token missing or license not accepted Accept the model license on huggingface.co and verify your token at $HF_HOME/token
    Container pull fails Login node resource limits Pull containers on a compute node via salloc
    NCCL error: unhandled cuda error GPU communication failure Ensure tensor_parallel_size matches --gpus-per-node and you are on a GPU partition
    Slow model loading on every run Downloading model weights each time Set HF_HOME to a persistent scratch location so the model is cached between runs
