    Apptainer Containers GPU Recipes

    Build and Run an Apptainer Container¶

    Package a custom software environment into an Apptainer container -- build a definition file, compile it on a compute node with --fakeroot, and run GPU-accelerated workloads with --nv.

    Goal¶

    Create an Apptainer container with PyTorch and custom libraries, build it on a compute node, and run a GPU job using the container.

    Prerequisites¶

    • An AICR account with a valid Slurm account (--account=ACCOUNT_NAME)
    • Familiarity with Containers (Apptainer) concepts
    • Knowledge of which Python packages and system libraries your project needs

    Steps¶

    1. Write a Definition File¶

    Create the definition file at /scratch/$USER/containers/pytorch_custom.def:

    $ mkdir -p /scratch/$USER/containers

    Bootstrap: docker
From: pytorch/pytorch:2.6.0-cuda12.6-cudnn9-runtime

%labels
    Author USERNAME
    Description PyTorch with custom scientific libraries

%post
    apt-get update && apt-get install -y --no-install-recommends \
        git \
        wget \
        libgl1-mesa-glx \
        libglib2.0-0 \
        && rm -rf /var/lib/apt/lists/*

    pip install --no-cache-dir \
        numpy \
        scipy \
        pandas \
        matplotlib \
        scikit-learn \
        transformers \
        datasets \
        accelerate \
        wandb \
        timm

%environment
    export LC_ALL=C
    export PYTHONDONTWRITEBYTECODE=1

%runscript
    python "$@"

%test
    python -c "import torch; print(f'PyTorch {torch.__version__}, CUDA available: {torch.cuda.is_available()}')"

    The sections do the following:

    Section Purpose
    Bootstrap / From Base image (PyTorch with CUDA from Docker Hub)
    %post Commands run during build (install system and Python packages)
    %environment Environment variables set at runtime
    %runscript Default command when container is executed
    %test Validation command run after build completes

    2. Build the Container on a Compute Node¶

    Container builds are CPU- and memory-intensive. Always build on a compute node, not a login node:

    $ salloc --partition=cpu --cpus-per-task=8 --mem=32G --time=01:00:00 --account=ACCOUNT_NAME

    Once on the compute node, build with --fakeroot:

    $ apptainer build --fakeroot /scratch/$USER/containers/pytorch_custom.sif /scratch/$USER/containers/pytorch_custom.def

    The build pulls the base Docker image, runs the %post commands, and produces a .sif file. This typically takes 5--15 minutes.

    Warning

    Build on a compute node, not a login node. Container builds use significant CPU and memory and will be terminated on login nodes.

    After the build completes, exit the interactive session:

    $ exit

    3. Test the Container Interactively¶

    Start an interactive GPU session and verify the container works:

    $ salloc --partition=rtx-devel --gpus=1 --cpus-per-task=4 --mem=16G --time=00:30:00 --account=ACCOUNT_NAME
$ apptainer exec --nv /scratch/$USER/containers/pytorch_custom.sif python -c "
import torch
print(f'PyTorch {torch.__version__}')
print(f'CUDA available: {torch.cuda.is_available()}')
print(f'GPU: {torch.cuda.get_device_name(0)}')

import transformers
print(f'Transformers {transformers.__version__}')
"
$ exit

    The --nv flag binds the host NVIDIA drivers and libraries into the container, giving your application GPU access.

    4. Write the Job Script¶

    Create container_job.sh:

    #!/bin/bash
#SBATCH --job-name=container-train
#SBATCH --partition=b200-batch
#SBATCH --gpus=1
#SBATCH --cpus-per-task=8
#SBATCH --mem=32G
#SBATCH --time=08:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

apptainer exec --nv \
    --bind /scratch/$USER:/scratch/$USER \
    --bind /work/INSTITUTION:/work/INSTITUTION \
    /scratch/$USER/containers/pytorch_custom.sif \
    python /scratch/$USER/my_project/train.py --epochs 50

    5. Submit the Job¶

    $ sbatch container_job.sh
Submitted batch job 12345

    Multi-GPU with Containers¶

    For multi-GPU jobs, use torchrun inside the container:

    #!/bin/bash
#SBATCH --job-name=container-multigpu
#SBATCH --partition=b200-batch
#SBATCH --nodes=1
#SBATCH --gpus-per-node=8
#SBATCH --cpus-per-task=64
#SBATCH --mem=0
#SBATCH --time=12:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

apptainer exec --nv \
    --bind /scratch/$USER:/scratch/$USER \
    /scratch/$USER/containers/pytorch_custom.sif \
    torchrun --nproc_per_node=8 /scratch/$USER/my_project/train.py

    Job Script¶

    The single-GPU container job script for reference:

    #!/bin/bash
#SBATCH --job-name=container-train
#SBATCH --partition=b200-batch
#SBATCH --gpus=1

#SBATCH --mem=32G

    Expected Output¶

    A successful build produces a .sif file:

    $ ls -lh /scratch/$USER/containers/pytorch_custom.sif
-rwxr-xr-x 1 USERNAME GROUP 6.2G Apr  4 10:30 pytorch_custom.sif

    Testing the container with --nv confirms GPU access:

    PyTorch 2.6.0
CUDA available: True
GPU: NVIDIA B200
Transformers 4.x.x

    Troubleshooting¶

    Problem Cause Fix
    FATAL: could not use fakeroot fakeroot not available on login node Build on a compute node via salloc, not on a login node
    Build fails during pip install Network timeout downloading packages Retry the build; add --retries 3 to the pip install line in the definition file
    CUDA not available at runtime Missing --nv flag Always use apptainer exec --nv for GPU workloads
    Permission denied accessing /scratch Directory not bound into container Add --bind /scratch/$USER:/scratch/$USER to the apptainer exec command
    Container is very large (>15 GB) Too many packages or build artifacts Use --no-cache-dir with pip; clean apt caches in %post with rm -rf /var/lib/apt/lists/*
    ImportError for a package at runtime Package not installed during build Add it to the pip install line in %post and rebuild
