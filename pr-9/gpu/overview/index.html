    B200 GPU InfiniBand RTX PRO 6000

    GPU Overview¶

    AICR has 400 GPUs across 50 nodes, organized into two Blackwell-generation GPU families connected by NDR400 InfiniBand.

    GPU Hardware¶

    GPU Model VRAM Memory Type Nodes GPUs/Node Total GPUs Partitions
    NVIDIA B200 192 GB HBM3e 31 (b-series) 8 248 b200-batch, b200-devel
    NVIDIA RTX PRO 6000 96 GB GDDR7 19 (a-series) 8 152 rtx-batch, rtx-devel

    Both GPUs are Blackwell architecture (compute capability 10.0, sm_100).

    B200 vs RTX PRO 6000¶

    Feature B200 RTX PRO 6000
    VRAM 192 GB HBM3e 96 GB GDDR7
    Architecture Blackwell (sm_100) Blackwell (sm_100)
    Memory bandwidth Very high (HBM3e) High (GDDR7)
    Dedicated IB fabric Yes (8 ports/node for NCCL) No (shared fabric)
    Best for Large-scale training, multi-node, large model inference General GPU compute, smaller training, single-node inference
    • B200: 192 GB VRAM per GPU means a single B200 can hold models up to ~90B parameters in BF16, or ~180B in 4-bit quantization. The dedicated compute InfiniBand fabric makes multi-node training fast.
    • RTX PRO 6000: 96 GB VRAM per GPU is substantial — enough for most single-GPU training workloads and models up to ~45B parameters in BF16.

    Tip

    If you don't know which to pick, start with rtx-devel for testing, then move to b200-batch for production training at scale.

    Compute Capability¶

    Both GPU types are compute capability 10.0 (sm_100). If you compile CUDA code, use:

    $ nvcc -arch=sm_100 my_program.cu

    PyTorch and JAX handle this automatically — you do not need to set architecture flags for Python-based workflows.

    Interconnect¶

    AICR has two physically separate InfiniBand NDR400 (400 Gb/s) fabrics:

    • Compute fabric — B200 nodes only (8 IB ports per node, one per GPU). Optimized for NCCL collective operations during distributed training.
    • Shared fabric — All node types. Carries NFS-RDMA storage traffic and general communication.

    This dual-fabric design means B200 distributed training does not compete with storage I/O for network bandwidth.

    MIG (Multi-Instance GPU)¶

    NVIDIA Multi-Instance GPU can partition a physical GPU into smaller, isolated instances. The initial AICR deployment does not use MIG, but this may change depending on workload patterns.

    GPU Driver and CUDA¶

    The NVIDIA datacenter driver (580.x series) is pre-installed on all GPU nodes. Load the CUDA toolkit as a module:

    $ module load cuda
