    Filesystem Overview¶

    AICR storage is separated into two categories: User storage and Project storage. Each user has both a Home and Scratch directory, and each project has a shared Work directory.

    Each of these have a different purpose, size, and characteristics. All AICR storage is fast flash storage.

    AICR storage is not backed up

    All storage on AICR is not backed up outside of snapshots in Home and Work. Any files that cannot be easily replaced should be backed up outside of AICR.

    AICR storage is meant to be ephemeral. Data should only exist on AICR when it is needed for active compute projects.

    • Home: Your Home directory is meant for your most important files, has snapshots. We recommend keeping your software and code in your home directory.
    • Scratch: Scratch space is meant for data used in actively running jobs. Scratch is meant for temporary files and is subject to regular purging. Scratch is not backed up and does not have snapshots and should not be used for long term storage.
    • Shared: Centralized datasets and software available to all users, maintained by AICR and institutional RC teams.
    • Work: Shared project space where the bulk of your project data should live. This storage is longer-term than Scratch, but should be cleared up at the end of the project lifecycle. Work also has snapshots.
    Storage Path Quota Snapshots Purge Best For
    Home /home/USERNAME 100 GiB 7-day None Code, configs, small environments
    Scratch /scratch/USERNAME ~10 TiB (~15M file limit) None Regular (~30 days) Job I/O, large files, checkpoints
    Shared /apps/aicr, /apps/<institution> Centrally managed 7-day None Software modules, shared datasets
    Work /work/INSTITUTION/PROJECTNAME Per-project (varies) 7-day None Project data, collaboration
