    Getting Started Quotas Storage

    Managing Quotas¶

    Each storage area on AICR has a quota. If you run out of space, jobs that write to that filesystem will fail. This page explains how to check your usage and free up space.

    Checking Usage¶

    Home Directory¶

    $ df -h ~

    This shows total, used, and available space for your home directory.

    For a detailed breakdown by subdirectory:

    $ du -sh ~/*/

    Scratch¶

    Scratch does not have per-user quota enforcement in the same way as Home, but it has a 10 TB allocation per user:

    $ du -sh /scratch/$USER

    Work¶

    Check your institution's Work directory usage:

    $ du -sh /work/INSTITUTION

    Replace INSTITUTION with your institution's directory name (e.g., mit, bu, yale, umass/umass).

    Finding Large Files¶

    When you need to free space, find what's taking the most room:

    $ du -ah /home/$USER | sort -rh | head -20

    This lists the 20 largest files and directories in your home.

    Tip

    If ncdu is available, it provides an interactive, navigable view of disk usage:

    $ ncdu /home/$USER

    Use arrow keys to navigate, d to delete, q to quit.

    Common Space Hogs¶

    Conda and pip Caches¶

    Conda environments and pip caches can grow large quickly, especially with ML frameworks:

    # Check conda cache size
$ du -sh ~/.conda/pkgs/

# Check pip cache size
$ du -sh ~/.cache/pip/

# Clean conda cache (removes unused packages)
$ conda clean --all

# Clean pip cache
$ pip cache purge

    Tip

    Create large Conda environments on scratch instead of home:

    $ conda create --prefix /scratch/$USER/conda-envs/ENVNAME python=3.12

    HuggingFace Model Cache¶

    HuggingFace downloads models to ~/.cache/huggingface/ by default, which can quickly fill your 100 GB home quota. Redirect the cache to scratch:

    $ export HF_HOME=/scratch/$USER/.cache/huggingface

    Add this line to your ~/.bashrc to make it permanent. See Hugging Face on AICR for more details.

    VS Code Server¶

    If you use VS Code Remote-SSH, extensions are stored in ~/.vscode-server/:

    $ du -sh ~/.vscode-server/

    Uninstall extensions you no longer use from within VS Code.

    Old Job Output¶

    Job output files (*.out, *.err) accumulate in the directory where you submitted jobs. Review and delete old output regularly.

    Freeing Space¶

    Strategies for reducing usage, from quickest to most thorough:

    1. Clean caches: conda clean --all, pip cache purge
    2. Redirect caches: Move HuggingFace, Conda, and pip caches to scratch (see above)
    3. Compress data: tar -czf archive.tar.gz directory/ then remove the original
    4. Delete old output: Remove job output files you no longer need
    5. Move large files to scratch: Container images (.sif), datasets, and checkpoints belong on scratch, not home
    6. Remove old environments: conda env remove --name ENVNAME

    Warning

    Do not delete files from /home/$USER/.ssh/ or /home/$USER/.bashrc unless you know what you are doing — these are essential for login and cluster access.
