    Data Transfer Getting Started Globus

    Transferring Data¶

    There are multiple approaches to moving files depending on your goals, the data volume, and your comfort level. This page covers different methods and inter-system transfers.

    Tip

    Use OnDemand for routine transfers and Globus for large files or bulk operations. Command-line tools like scp or rsync are available for those preferring terminal workflows.

    OnDemand¶

    AICR offers an Open OnDemand Portal for file management via a graphical File Browser. Capabilities include:

    • Upload and download files and directories
    • Rename, move, and delete files and directories
    • View and edit files in-place

    After logging in, select Files > Home Directory. Drag-and-drop between your File Browser and desktop file explorer windows. Multi-select files using Control/Command+click, then use the Download/Upload buttons.

    Viewing and Editing Files¶

    Click a file, then select "View" or "Edit" buttons. The editor provides line numbers, syntax highlighting for most languages, adjustable display colors, font sizing, and word-wrap options. Click "Save" to persist changes.

    Globus¶

    For multiple files or particularly large datasets, Globus facilitates high-volume transfers between systems. Collections serve as Globus access mechanisms.

    Advantages of Globus:

    • Simple web-based transfer initiation
    • No need to remain logged in during transfers
    • Automatic resume on reconnection

    Transfer Steps¶

    1. Log in to Globus using your institutional credentials
    2. Select collections for source and destination. Search for AICR Collection — this is the single shared Globus collection for all AICR users, landing in your home directory by default. Use Globus Connect Personal for your local computer.
    3. Navigate directories on both sides; select files/folders to transfer
    4. Configure options via "Transfer and Timer Options" (syncing, scheduling)
    5. Start transfer by pressing the source-side "Start" button
    6. Monitor via "Activity" tab; email notifications confirm completion or issues

    Command Line¶

    Common tools: scp and rsync. Run these from your local system before connecting to AICR.

    Requirements:

    • Full remote path to target directory
    • SSH access to the remote system

    Both commands follow the pattern: specify source, then destination, with remote paths prefixed by hostname:.

    scp¶

    Open a terminal on your local computer (not connected to AICR).

    Local to AICR:

    $ scp LOCAL_FILE USERNAME@transfer.aicr.ai:PATH_TO_AICR_DIR

    Example:

    $ scp myscript.py USERNAME@transfer.aicr.ai:/home/USERNAME/mycode/

    AICR to Local:

    $ scp USERNAME@transfer.aicr.ai:PATH_TO_AICR_FILE PATH_TO_LOCAL_DIR

    Example:

    $ scp USERNAME@transfer.aicr.ai:/home/USERNAME/output/results.csv .

    The . represents the current directory.

    Recursive transfers (entire directories):

    $ scp -r LOCAL_DIR USERNAME@transfer.aicr.ai:PATH_TO_AICR_DIR

    rsync¶

    Similar to scp, but only transfers files differing between source and destination by default. Useful for syncing updates and resuming interrupted transfers.

    Basic usage: replace scp with rsync in the examples above.

    Common flags:

    • -r, --recursive — recursively copy subdirectories
    • -l, --links — preserve symbolic links
    • -u, --update — skip destination files with later timestamps
    • -v, --verbose — detailed output (add more v's for verbosity)
    • --partial — retain partially transferred files for resumption
    • --progress — display transfer progress
    • -n, --dry-run — preview actions without executing

    Run rsync --help for complete flag documentation.

    Moving Files Between AICR and Another Cluster¶

    SSH into one system, then initiate the transfer to the other. For example, transferring from your institution's cluster to AICR:

    $ ssh USERNAME@OTHER_CLUSTER_LOGIN
$ scp PATH_TO_FILE USERNAME@transfer.aicr.ai:PATH_TO_AICR_DIR

    Alternatively, SSH into AICR and pull the file from the other cluster.

    Graphical Applications for File Transfer¶

    Third-party tools offering drag-and-drop functionality:

    • Cyberduck (Mac, Windows)
    • FileZilla (Mac, Windows, Linux)
    • WinSCP (Windows only)

    Connect using the AICR data transfer node hostname.
