    Hugging Face on AICR¶

    Hugging Face provides a large ecosystem of pre-trained models, datasets, and training utilities. This page covers how to configure the cache, download models, run inference with pipelines, fine-tune with the Trainer API, and scale to multiple GPUs with Accelerate on AICR.

    Setting Up the Cache¶

    By default, Hugging Face downloads models and datasets to ~/.cache/huggingface, which counts against your 100 GB home quota. Large models (7B+ parameters) can be 15--140 GB each. Redirect the cache to scratch:

    Add to your ~/.bashrc:

    export HF_HOME=/scratch/$USER/.cache/huggingface

    Then reload:

    $ source ~/.bashrc

    Warning

    Files on /scratch are not backed up and may be subject to purge policies. Keep a record of which models you have downloaded so you can re-download them if needed.

    Downloading Models¶

    Small Models (Login Node)¶

    For small models (under a few GB), downloading on the login node is acceptable:

    $ pip install huggingface_hub
$ huggingface-cli download bert-base-uncased

    Large Models (Compute Node)¶

    For large models, download on a compute node to avoid overloading the login node. Request a short interactive session:

    $ salloc --partition=cpu --time=02:00:00 --mem=32G --account=ACCOUNT_NAME

    Then download:

    $ huggingface-cli download meta-llama/Llama-3.1-70B-Instruct

    Tip

    For gated models (e.g., Llama), you need a Hugging Face access token. Create one at huggingface.co/settings/tokens and log in:

    $ huggingface-cli login

    The token is stored in $HF_HOME/token. Do not share it or commit it to version control.

    Downloading Specific Files¶

    If you only need certain files from a model (e.g., just the tokenizer or a specific checkpoint):

    $ huggingface-cli download meta-llama/Llama-3.1-8B-Instruct --include "tokenizer*"

    Pipeline API (Quick Inference)¶

    The pipeline API is the fastest way to run inference with a pre-trained model:

    from transformers import pipeline

classifier = pipeline("sentiment-analysis", device=0)
result = classifier("AICR is a great cluster for AI research.")
print(result)
# [{'label': 'POSITIVE', 'score': 0.9998}]

    For text generation:

    from transformers import pipeline

generator = pipeline(
    "text-generation",
    model="meta-llama/Llama-3.1-8B-Instruct",
    device_map="auto",
    torch_dtype="bfloat16",
)

output = generator("Explain what a GPU is in one sentence:", max_new_tokens=50)
print(output[0]["generated_text"])

    Note

    Setting device_map="auto" distributes the model across all available GPUs automatically. This is convenient but less efficient than explicit tensor parallelism via vLLM. For production inference, see Inference and Serving.

    Fine-Tuning with Trainer¶

    The Hugging Face Trainer class handles training loops, logging, checkpointing, and evaluation. Here is a job script for fine-tuning a model on AICR:

    #!/bin/bash
#SBATCH --job-name=hf-finetune
#SBATCH --partition=b200-batch
#SBATCH --gpus=1
#SBATCH --cpus-per-task=8
#SBATCH --mem=64G
#SBATCH --time=12:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge
source ~/hf-env/bin/activate

export HF_HOME=/scratch/$USER/.cache/huggingface

python finetune.py

    Example finetune.py:

    import os
from transformers import (
    AutoModelForSequenceClassification,
    AutoTokenizer,
    Trainer,
    TrainingArguments,
)
from datasets import load_dataset

model_name = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=2)

dataset = load_dataset("imdb")

def tokenize(batch):
    return tokenizer(batch["text"], padding="max_length", truncation=True, max_length=512)

tokenized = dataset.map(tokenize, batched=True)

training_args = TrainingArguments(
    output_dir=os.path.expandvars("/scratch/$USER/hf-checkpoints/imdb"),
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=64,
    eval_strategy="epoch",
    save_strategy="epoch",
    logging_steps=100,
    bf16=True,
    dataloader_num_workers=4,
    dataloader_pin_memory=True,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized["train"],
    eval_dataset=tokenized["test"],
)

trainer.train()
trainer.save_model(os.path.expandvars("/scratch/$USER/hf-models/imdb-classifier"))

    Multi-GPU Training with Accelerate¶

    Hugging Face Accelerate is a lightweight library that enables multi-GPU training with minimal code changes. It works with both the Trainer API and custom training loops.

    Configure Accelerate¶

    Generate a configuration file:

    $ accelerate config

    Answer the prompts for your setup (multi-GPU, single node, BF16, etc.). The configuration is saved to ~/.cache/huggingface/accelerate/default_config.yaml.

    Multi-GPU Job Script with Accelerate¶

    #!/bin/bash
#SBATCH --job-name=hf-accelerate
#SBATCH --partition=b200-batch
#SBATCH --nodes=1
#SBATCH --gpus-per-node=8
#SBATCH --cpus-per-task=64
#SBATCH --mem=0
#SBATCH --time=12:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge
source ~/hf-env/bin/activate

export HF_HOME=/scratch/$USER/.cache/huggingface

accelerate launch \
    --num_processes=8 \
    --mixed_precision=bf16 \
    finetune.py

    Tip

    The Trainer class automatically detects Accelerate and uses it for distributed training. You can use the same finetune.py script shown above -- Trainer handles multi-GPU data distribution and gradient synchronization.

    Multi-Node with Accelerate¶

    For multi-node training, combine srun with Accelerate:

    #!/bin/bash
#SBATCH --job-name=hf-multinode
#SBATCH --partition=b200-batch
#SBATCH --nodes=2
#SBATCH --gpus-per-node=8
#SBATCH --ntasks-per-node=1
#SBATCH --cpus-per-task=64
#SBATCH --mem=0
#SBATCH --time=24:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge
source ~/hf-env/bin/activate

export HF_HOME=/scratch/$USER/.cache/huggingface
export MASTER_ADDR=$(scontrol show hostnames $SLURM_JOB_NODELIST | head -n 1)
export MASTER_PORT=29500

srun accelerate launch \
    --num_processes=16 \
    --num_machines=2 \
    --mixed_precision=bf16 \
    --main_process_ip=$MASTER_ADDR \
    --main_process_port=$MASTER_PORT \
    finetune.py

    Cache Management¶

    Over time, cached models and datasets accumulate. Monitor and clean up:

    # Check cache size
$ du -sh /scratch/$USER/.cache/huggingface/

# List cached models
$ huggingface-cli scan-cache

# Delete specific cached models
$ huggingface-cli delete-cache

    Tip

    The huggingface-cli scan-cache command shows each cached repo, its size, and last access time. Use huggingface-cli delete-cache interactively to select which repos to remove.

    Sharing Models Across a Group¶

    If multiple members of your research group use the same models, consider downloading once to a shared location:

    $ export HF_HOME=/scratch/GROUP_NAME/shared-hf-cache
$ huggingface-cli download meta-llama/Llama-3.1-70B-Instruct

    Then each group member can point HF_HOME to the shared cache. Coordinate to avoid duplicate downloads.
