    GPU Profiling¶

    If your GPU utilization is low or training is slower than expected, profiling helps identify the bottleneck.

    Quick Check: nvidia-smi¶

    The fastest way to see if your GPU is being used:

    $ nvidia-smi

    Key metrics:

    Metric Healthy Range Problem If
    GPU-Util 80–100% Below 50% consistently
    Memory-Usage Depends on model Near 0% means GPU isn't being used
    Temperature Below 80°C Above 85°C may indicate throttling

    For continuous monitoring:

    $ watch -n 1 nvidia-smi

    Tip

    Run this in a second terminal while your job is running (SSH to the compute node — see Monitoring Jobs).

    Common Bottlenecks¶

    Symptom Likely Cause Fix
    GPU-Util 0–10% Data loading is the bottleneck Increase num_workers in DataLoader, enable pin_memory
    GPU-Util spikes and drops CPU-GPU data transfer stalls Use pin_memory=True, increase prefetch_factor
    GPU-Util ~50% Batch size too small Increase batch size to better fill the GPU
    High GPU-Util, slow training Model is compute-bound (expected) Enable mixed precision (torch.amp) or torch.compile
    GPU memory near 100% Out of memory risk Reduce batch size, enable gradient checkpointing

    PyTorch Profiler¶

    For detailed analysis, use PyTorch's built-in profiler:

    import torch
from torch.profiler import profile, record_function, ProfilerActivity

with profile(
    activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA],
    record_shapes=True,
    with_stack=True,
) as prof:
    for step, batch in enumerate(train_loader):

            break

print(prof.key_averages().table(sort_by="cuda_time_total", row_limit=20))

    This shows which operations take the most GPU time.

    Viewing Traces with TensorBoard¶

    Export the profiler trace for visualization:

    import os
prof.export_chrome_trace(os.path.expandvars("/scratch/$USER/profiler_trace.json"))

    Then view with TensorBoard:

    $ pip install tensorboard torch-tb-profiler
$ tensorboard --logdir=/scratch/$USER/ --port=6006 --bind_all

    Access via SSH tunnel: ssh -L 6006:NODE_NAME:6006 USERNAME@login.aicr.ai

    Profiling on Devel Partitions¶

    Use devel partitions (4h max) for profiling sessions. This avoids tying up batch resources for exploratory work:

    $ salloc --partition=rtx-devel --gpus=1 --cpus-per-task=8 --mem=32G --time=02:00:00 --account=ACCOUNT_NAME
