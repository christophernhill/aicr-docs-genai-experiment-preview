    AI/ML GPU Julia SciML

    Julia and SciML¶

    Julia is well suited for scientific machine learning (SciML), combining the expressiveness of a high-level language with the performance of compiled code. AICR provides Julia through the module system with GPU support via CUDA.jl.

    Loading Julia¶

    Load Julia via the module system:

    $ module spider julia
$ module load julia
$ julia --version

    Package Management with Pkg.jl¶

    Julia uses Pkg.jl for dependency management. Every project should have its own environment defined by Project.toml and Manifest.toml files.

    Setting JULIA_DEPOT_PATH¶

    Julia stores downloaded packages, compiled artifacts, and registries in its depot directory (defaults to ~/.julia). This can quickly consume your 100 GB home quota. Redirect it to scratch:

    $ echo 'export JULIA_DEPOT_PATH=/scratch/$USER/.julia' >> ~/.bashrc
$ source ~/.bashrc

    Warning

    Set JULIA_DEPOT_PATH before installing any packages. Moving an existing depot can cause path mismatches in compiled artifacts.

    Creating a Project Environment¶

    $ mkdir -p /scratch/$USER/my_project
$ cd /scratch/$USER/my_project
$ julia --project=.

    Inside the Julia REPL:

    using Pkg
Pkg.add("DifferentialEquations")
Pkg.add("Lux")
Pkg.add("CUDA")

    This creates Project.toml and Manifest.toml files in the project directory. Always commit these files to version control for reproducibility.

    Activating an Existing Environment¶

    $ julia --project=/scratch/$USER/my_project

    Or from the Julia REPL:

    using Pkg
Pkg.activate("/scratch/$USER/my_project")
Pkg.instantiate()  # install all packages from Manifest.toml

    The SciML Ecosystem¶

    The SciML ecosystem is a collection of Julia packages for scientific computing and machine learning. Key packages include:

    Package Purpose
    DifferentialEquations.jl Solvers for ODEs, SDEs, PDEs, DAEs
    ModelingToolkit.jl Symbolic modeling and automatic equation simplification
    Lux.jl Neural network framework (SciML-native)
    Optimization.jl Unified optimization interface
    SciMLSensitivity.jl Adjoint methods for differentiating through solvers
    NeuralPDE.jl Physics-informed neural networks (PINNs)

    DifferentialEquations.jl¶

    DifferentialEquations.jl provides high-performance solvers for a wide range of differential equations:

    using DifferentialEquations

function lorenz!(du, u, p, t)
    σ, ρ, β = p
    du[1] = σ * (u[2] - u[1])
    du[2] = u[1] * (ρ - u[3]) - u[2]
    du[3] = u[1] * u[2] - β * u[3]
end

u0 = [1.0, 0.0, 0.0]
tspan = (0.0, 100.0)
p = [10.0, 28.0, 8/3]

prob = ODEProblem(lorenz!, u0, tspan, p)
sol = solve(prob, Tsit5())

    ModelingToolkit.jl¶

    ModelingToolkit.jl allows you to define models symbolically and let the compiler optimize the resulting system:

    using ModelingToolkit
using ModelingToolkit: t_nounits as t, D_nounits as D

@variables x(t) y(t) z(t)
@parameters σ ρ β

eqs = [
    D(x) ~ σ * (y - x),
    D(y) ~ x * (ρ - z) - y,
    D(z) ~ x * y - β * z,
]

@named lorenz = ODESystem(eqs, t)
lorenz = structural_simplify(lorenz)

    Neural ODEs with Lux.jl¶

    Lux.jl is the SciML ecosystem's neural network framework, designed for scientific machine learning with explicit parameter handling.

    A neural ODE combines a neural network with a differential equation solver:

    using Lux, DifferentialEquations, SciMLSensitivity
using Zygote, Optimisers, Random

# Define the neural network
model = Chain(
    Dense(2 => 32, tanh),
    Dense(32 => 32, tanh),
    Dense(32 => 2),
)

rng = Random.default_rng()
ps, st = Lux.setup(rng, model)

# Neural ODE dynamics
function neural_ode!(du, u, p, t)
    du .= first(model(u, p, st))
end

# Solve and train
prob = ODEProblem(neural_ode!, u0, tspan, ps)

function loss(p)
    pred = solve(prob, Tsit5(); p=p, saveat=0.1)
    sum(abs2, pred .- target_data)
end

    GPU Computing with CUDA.jl¶

    CUDA.jl provides GPU acceleration for Julia. Load the CUDA module alongside Julia:

    $ module load cuda julia

    Verify GPU access in Julia:

    using CUDA
CUDA.functional()    # should return true
CUDA.devices()       # list available GPUs
CUDA.versioninfo()   # CUDA toolkit and driver info

    Move arrays to the GPU with CuArray:

    using CUDA

a = CUDA.randn(1000, 1000)
b = CUDA.randn(1000, 1000)
c = a * b  # runs on GPU

    GPU-Accelerated Differential Equations¶

    DifferentialEquations.jl can solve ensembles of problems on the GPU:

    using DifferentialEquations, CUDA

prob = ODEProblem(lorenz!, u0, tspan, p)
ensemble_prob = EnsembleProblem(prob)

sol = solve(
    ensemble_prob, Tsit5(),
    EnsembleGPUKernel(CUDA.CUDABackend()),
    trajectories = 10_000,
)

    Note

    GPU-accelerated ensemble solves are most effective when you need to solve many independent instances of the same problem (e.g., parameter sweeps, uncertainty quantification).

    Batch Job Script¶

    Submit a Julia SciML job on a GPU partition:

    #!/bin/bash
#SBATCH --job-name=julia_sciml
#SBATCH --partition=rtx-batch
#SBATCH --gpus=1
#SBATCH --cpus-per-task=8
#SBATCH --mem=32G
#SBATCH --time=08:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge
module load cuda julia

export JULIA_DEPOT_PATH=/scratch/$USER/.julia

julia --project=/scratch/$USER/my_project train.jl

    Submit with:

    $ sbatch julia_sciml.sh

    Tip

    Julia has a noticeable compilation delay the first time code runs ("time to first plot"). For production jobs, consider using PackageCompiler.jl to create a custom system image with your dependencies precompiled.

    Multi-Threading¶

    Julia supports multi-threading natively. Set the number of threads to match your Slurm allocation:

    #!/bin/bash
#SBATCH --job-name=julia_threaded
#SBATCH --partition=cpu
#SBATCH --cpus-per-task=16
#SBATCH --mem=32G
#SBATCH --time=04:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge
module load julia

export JULIA_DEPOT_PATH=/scratch/$USER/.julia
export JULIA_NUM_THREADS=$SLURM_CPUS_PER_TASK

julia --project=/scratch/$USER/my_project --threads=$SLURM_CPUS_PER_TASK run.jl

    Inside your Julia code, use Threads.@threads for parallel loops:

    using Base.Threads

results = zeros(1000)
@threads for i in 1:1000
    results[i] = expensive_computation(i)
end

println("Using $(nthreads()) threads")

    Warning

    Do not set JULIA_NUM_THREADS higher than --cpus-per-task. Oversubscribing CPU cores causes contention and slows your job.
