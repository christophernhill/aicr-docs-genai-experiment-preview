    Reproducible Research¶

    Reproducibility is essential for credible science. On a shared cluster like AICR, where software stacks, hardware, and scheduling vary, extra care is needed to ensure results can be recreated. This page covers practical techniques for making your AI research reproducible.

    Set Random Seeds¶

    Most AI frameworks use random number generators for weight initialization, data shuffling, and augmentation. Set seeds explicitly to get deterministic results:

    PyTorch¶

    import torch
import random
import numpy as np

SEED = 42

random.seed(SEED)
np.random.seed(SEED)
torch.manual_seed(SEED)
torch.cuda.manual_seed_all(SEED)

# For full determinism (may reduce performance)
torch.backends.cudnn.deterministic = True
torch.backends.cudnn.benchmark = False

    TensorFlow¶

    import tensorflow as tf
import random
import numpy as np

SEED = 42

random.seed(SEED)
np.random.seed(SEED)
tf.random.set_seed(SEED)

    JAX¶

    import jax

key = jax.random.PRNGKey(42)

    Note

    Full determinism on GPUs is not always possible. CUDA operations like atomicAdd introduce non-determinism by design. Setting seeds gets you close, but identical results across different GPU hardware or driver versions are not guaranteed.

    Export Your Environment¶

    Capture the exact package versions used in your experiment. This is the single most important step for reproducibility after setting seeds.

    Conda¶

    $ conda env export --no-builds > environment.yml

    Recreate later:

    $ conda env create -f environment.yml

    Pip¶

    $ pip freeze > requirements.txt

    Recreate later:

    $ pip install -r requirements.txt

    Tip

    Export your environment at the time of the experiment, not after. Installing additional packages later changes the output of pip freeze.

    Module List¶

    Record which AICR modules were loaded:

    $ module list 2>&1 | tee modules-loaded.txt

    Include module load commands in your Slurm job scripts so the environment is reconstructed automatically on each run.

    Use Containers for Full Reproducibility¶

    Conda and pip environments capture Python packages but not system libraries, CUDA versions, or OS-level dependencies. Containers capture everything.

    Build a Definition File¶

    Create a definition file that specifies the complete software stack:

    Bootstrap: docker
From: nvcr.io/nvidia/pytorch:RELEASE_TAG

%post
    pip install transformers datasets wandb

%environment
    export PYTHONUNBUFFERED=1

    Build and Run¶

    $ apptainer build my_env.sif my_env.def

    #!/bin/bash
#SBATCH --job-name=container-job
#SBATCH --partition=rtx-batch
#SBATCH --gpus=1
#SBATCH --mem=32G
#SBATCH --time=04:00:00
#SBATCH --account=ACCOUNT_NAME

apptainer exec --nv my_env.sif python train.py

    Store your .def files in version control alongside your code. The .sif image can be rebuilt from the definition at any time.

    See Containers for complete details on building and using containers on AICR.

    Version Control with Git¶

    Track your code, job scripts, configuration files, and environment files in a git repository:

    $ git init /home/$USER/PROJECT_NAME
$ cd /home/$USER/PROJECT_NAME
$ git add train.py environment.yml job.sh
$ git commit -m "Initial training setup"

    What to Track¶

    Track Do Not Track
    Training scripts Large datasets
    Job submission scripts Model checkpoints
    Configuration files (hyperparameters) Logs and output files
    environment.yml / requirements.txt .sif container images
    Container .def files Scratch data
    Analysis notebooks

    Tag Experiments¶

    Tag or branch each experiment so you can return to the exact code that produced a result:

    $ git tag -a exp-001 -m "Baseline ResNet50, lr=0.001, batch=64"
$ git push origin exp-001

    Store Job Scripts in Slurm Accounting¶

    Slurm records the job script for every submitted job. You can retrieve it later:

    $ sacct -j JOBID --format=JobID,JobName,Submit,Start,End,State

    For a complete record, keep your job scripts in your git repository alongside the code they execute. This creates a self-contained record of what ran, when, and how.

    Reproducibility Checklist for Each Experiment¶

    1. Set random seeds in your training script.
    2. Export your environment (conda env export or pip freeze).
    3. Record loaded modules in the job script.
    4. Commit all code and configs to git before submitting the job.
    5. Tag the commit with the experiment identifier.
    6. Log the Slurm job ID alongside your results.

    Log Hyperparameters and Metrics¶

    Use an experiment tracking tool to record hyperparameters, metrics, and artifacts automatically:

    • Weights & Biases (wandb) -- pip install wandb
    • MLflow -- pip install mlflow
    • TensorBoard -- included with PyTorch and TensorFlow
    import wandb

wandb.init(project="PROJECT_NAME", config={
    "learning_rate": 0.001,
    "batch_size": 64,
    "epochs": 50,
    "seed": 42,
})

    See Experiment Tracking for setup instructions on AICR.
