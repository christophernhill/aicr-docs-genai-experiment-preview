    Support Troubleshooting

    FAQ¶

    Frequently asked questions about using AICR.

    Login and Access¶

    I cannot log in. What should I do?¶

    AICR uses SSH certificate authentication. Check that your certificate files are in ~/.ssh/ with correct permissions:

    $ chmod 600 ~/.ssh/id_ed25519_aicr
$ chmod 644 ~/.ssh/id_ed25519_aicr-cert.pub

    If your certificate has expired, regenerate it through the OnDemand portal at ood.aicr.ai. See Login Troubleshooting for detailed steps.

    How do I get an AICR account?¶

    AICR accounts are provisioned through your institution's research computing team. See Requesting Access for the process at each institution.

    I can log in but my jobs fail immediately. What is wrong?¶

    Check that you are specifying --account=ACCOUNT_NAME in your job submission. Jobs without a valid account are rejected:

    $ sbatch --account=ACCOUNT_NAME my_job.sh

    See the Slurm Basics page for valid account names.

    Storage and Quotas¶

    I am over quota. How do I fix it?¶

    Check which filesystem is full:

    $ df -h /home/$USER
$ df -h /scratch/$USER

    Find the largest directories:

    $ du -sh /home/$USER/* | sort -rh | head -20

    Common space hogs include conda environments, pip caches, and old checkpoints. Remove what you no longer need:

    $ rm -rf /home/$USER/.cache/pip
$ conda clean --all

    See Managing Quotas for more cleanup strategies.

    Where should I store my data?¶

    Data Type Location
    Code, scripts, configs /home/$USER
    Active job data, checkpoints /scratch/$USER
    Shared project data /work/INSTITUTION

    See Data Management for the full workflow.

    Can I recover a deleted file?¶

    Files in /home/$USER and /work/INSTITUTION are protected by 7-day snapshots. Scratch has no snapshots. See Restoring from Snapshots.

    GPU Access¶

    How do I request a GPU?¶

    Add --gpus=N to your Slurm job and specify a GPU partition:

    $ salloc --partition=rtx-devel --gpus=1 --account=ACCOUNT_NAME

    Or in a batch script:

    #SBATCH --partition=rtx-batch
#SBATCH --gpus=1
#SBATCH --account=ACCOUNT_NAME

    What GPU types are available?¶

    Partition GPU Model
    rtx-batch, rtx-devel NVIDIA RTX PRO 6000
    b200-batch, b200-devel NVIDIA B200

    See GPU Overview for hardware specifications.

    My GPU job runs but nvidia-smi shows 0% utilization. Why?¶

    Common causes:

    • Data loading bottleneck. Your DataLoader is too slow. Increase num_workers or prefetch data.
    • Small batch size. The GPU finishes each batch quickly and waits for the next.
    • CPU-bound preprocessing. Move preprocessing into the DataLoader or preprocess offline.

    See GPU Best Practices and GPU Profiling.

    Software and Modules¶

    How do I find available software?¶

    Use the module system:

    $ module avail
$ module avail cuda
$ module spider PACKAGE_NAME

    See Modules for full details.

    How do I use a specific version of CUDA?¶

    $ module avail cuda
$ module load cuda/VERSION

    Can I install my own Python packages?¶

    Yes. Create a conda or pip virtual environment in your home directory:

    $ module load miniforge
$ conda create -n myenv python=3.11
$ conda activate myenv
$ pip install PACKAGE_NAME

    See Python on AICR.

    Jobs and Scheduling¶

    My job has been pending for a long time. Why?¶

    Check the reason:

    $ squeue -u $USER --format="%.18i %.9P %.30j %.8T %.10M %.6D %R"

    Common reasons:

    Reason Meaning
    Priority Other jobs have higher fairshare priority. Your job will run when resources free up.
    Resources Not enough free nodes matching your request right now.
    QOSMaxJobsPerUserLimit You have hit the concurrent job limit on devel partitions (4 jobs).
    AssocGrpBillingMinutes Your account has exceeded its allocation. Contact your institutional support team.

    See Efficient Resource Usage for tips on improving priority.

    How do I transfer files to and from AICR?¶

    For small transfers, use scp or rsync:

    $ rsync -avz LOCAL_FILE transfer.aicr.ai:/scratch/$USER/
$ scp transfer.aicr.ai:/scratch/$USER/RESULTS_FILE ./

    For large transfers (over 100 GB), use Globus for better performance and resumability. See Transferring Data.

    How long can my job run?¶

    Partition Max Wall Time
    Batch (rtx-batch, b200-batch) 24 hours
    Devel (rtx-devel, b200-devel) 4 hours
    CPU (cpu) 24 hours

    For jobs that need more time, implement checkpointing and use Slurm's --requeue feature.
