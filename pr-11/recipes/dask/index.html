    Parallel Data Analysis with Dask¶

    Process large datasets in parallel on AICR's CPU partition using Dask Distributed.

    Goal¶

    Set up a Dask distributed cluster within a Slurm job on the cpu partition and process a large Parquet dataset in parallel across multiple workers.

    Prerequisites¶

    • An AICR account with a valid Slurm account (--account=ACCOUNT_NAME)
    • A dataset (CSV or Parquet) staged on /scratch/$USER or /work/INSTITUTION

    Steps¶

    1. Create a Conda Environment¶

    $ conda create --prefix /scratch/$USER/conda-envs/dask python=3.12 -y
$ conda activate /scratch/$USER/conda-envs/dask
$ pip install dask[complete] distributed pyarrow fastparquet

    2. Prepare Example Data¶

    If you do not already have a dataset, generate a sample Parquet file for testing. Create /scratch/$USER/dask-demo/generate_data.py:

    """Generate a sample Parquet dataset for Dask demo."""
import os

import numpy as np
import pandas as pd

output_dir = os.path.expandvars("/scratch/$USER/dask-demo/data")
os.makedirs(output_dir, exist_ok=True)

rng = np.random.default_rng(42)
n_rows = 10_000_000

df = pd.DataFrame({
    "sensor_id": rng.integers(0, 1000, size=n_rows),
    "timestamp": pd.date_range("2025-01-01", periods=n_rows, freq="s"),
    "temperature": rng.normal(25.0, 5.0, size=n_rows),
    "pressure": rng.normal(1013.0, 10.0, size=n_rows),
    "humidity": rng.uniform(30.0, 90.0, size=n_rows),
})

output_path = os.path.join(output_dir, "sensor_readings.parquet")
df.to_parquet(output_path, engine="pyarrow", index=False)
print(f"Wrote {n_rows:,} rows to {output_path}")

    Run on a login node or interactive session:

    $ python /scratch/$USER/dask-demo/generate_data.py
Wrote 10,000,000 rows to /scratch/USERNAME/dask-demo/data/sensor_readings.parquet

    3. Write the Dask Analysis Script¶

    Create /scratch/$USER/dask-demo/dask_analysis.py:

    """Parallel data analysis with Dask Distributed."""
import os
import time

import dask.dataframe as dd
from dask.distributed import Client


def main():
    # Start a Dask client using the local cluster (workers spawned by Dask)
    n_workers = int(os.environ.get("SLURM_CPUS_PER_TASK", 4))
    client = Client(n_workers=n_workers, threads_per_worker=1)
    print(f"Dask dashboard: {client.dashboard_link}")
    print(f"Workers: {n_workers}")

    # Load data
    data_path = os.path.expandvars("/scratch/$USER/dask-demo/data/sensor_readings.parquet")
    df = dd.read_parquet(data_path)

    print(f"Columns: {list(df.columns)}")
    print(f"Partitions: {df.npartitions}")

    start = time.time()

    # --- Analysis: per-sensor statistics ---
    stats = df.groupby("sensor_id").agg({
        "temperature": ["mean", "std", "min", "max"],
        "pressure": ["mean", "std"],
        "humidity": ["mean"],
    }).compute()

    elapsed = time.time() - start
    print(f"\nGroupBy completed in {elapsed:.1f}s")
    print(f"Unique sensors: {len(stats)}")

    stats.columns = ["_".join(col) for col in stats.columns]
    top5 = stats.nlargest(5, "temperature_mean")
    print(f"\nTop 5 sensors by mean temperature:")
    print(top5[["temperature_mean", "temperature_std"]].to_string())

    # --- Analysis: time-based resampling ---
    start = time.time()
    df["timestamp"] = dd.to_datetime(df["timestamp"])
    hourly = df.set_index("timestamp").resample("1h").agg({
        "temperature": "mean",
        "pressure": "mean",
    }).compute()

    elapsed = time.time() - start
    print(f"\nHourly resample completed in {elapsed:.1f}s")
    print(f"Hourly records: {len(hourly)}")

    # Save results
    output_dir = os.path.expandvars("/scratch/$USER/dask-demo/results")
    os.makedirs(output_dir, exist_ok=True)
    stats.to_csv(os.path.join(output_dir, "sensor_stats.csv"))
    hourly.to_csv(os.path.join(output_dir, "hourly_averages.csv"))
    print(f"\nResults saved to {output_dir}")

    client.close()


if __name__ == "__main__":
    main()

    Note

    The $USER in Python path strings will not expand automatically. The script uses os.path.expandvars() to handle this. Alternatively, pass paths via environment variables in the job script and use os.environ["VAR_NAME"] in Python.

    4. Write the Job Script¶

    Create dask_job.sh:

    #!/bin/bash
#SBATCH --job-name=dask-analysis
#SBATCH --partition=cpu
#SBATCH --nodes=1
#SBATCH --cpus-per-task=32
#SBATCH --mem=64G
#SBATCH --time=02:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --output=%x-%j.out

module purge

eval "$(conda shell.bash hook)"
conda activate /scratch/$USER/conda-envs/dask

python /scratch/$USER/dask-demo/dask_analysis.py

    5. Submit the Job¶

    $ sbatch dask_job.sh
Submitted batch job 12345

    Troubleshooting¶

    Problem Cause Fix
    MemoryError during .compute() Result too large for available RAM Increase --mem, reduce n_workers, or write results incrementally with .to_parquet() instead of .compute()
    Dask uses only 1 worker SLURM_CPUS_PER_TASK not set or not read Ensure --cpus-per-task is specified in your Slurm script
    Slow performance Data on home directory Stage data to /scratch/$USER for faster I/O; use Parquet instead of CSV
    FileNotFoundError for data path $USER not expanded in Python string Use os.path.expandvars() for any path containing $USER
    ImportError: No module named 'dask' Conda environment not activated in the job Add eval "$(conda shell.bash hook)" and conda activate before the python command
    Dashboard inaccessible from local machine Running on compute node Use an SSH tunnel: ssh -L 8787:NODE_NAME:8787 USERNAME@login.aicr.ai
