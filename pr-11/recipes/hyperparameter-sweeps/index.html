    Hyperparameter Sweeps¶

    Combine Slurm job arrays with Weights & Biases (W&B) for systematic hyperparameter tuning across many GPU jobs.

    Goal¶

    Run 20 training experiments with different learning rates using a Slurm job array (--array=0-19), log results to W&B, and identify the best configuration.

    Prerequisites¶

    • An AICR account with a valid Slurm account (--account=ACCOUNT_NAME)
    • A Conda environment with PyTorch and wandb installed
    • A W&B account and API key (see Experiment Tracking)
    • Familiarity with Job Arrays

    Steps¶

    1. Set Up the Environment¶

    $ conda create --prefix /scratch/$USER/conda-envs/sweep python=3.12 -y
$ conda activate /scratch/$USER/conda-envs/sweep
$ conda install pytorch torchvision pytorch-cuda=12.4 -c pytorch -c nvidia -y
$ pip install wandb

    2. Configure W&B Authentication¶

    Store your API key securely:

    $ echo "YOUR_API_KEY" > ~/.wandb_api_key
$ chmod 600 ~/.wandb_api_key

    Warning

    Never hardcode your API key in job scripts or source code. Use the file-based approach above.

    3. Write the Training Script with W&B Logging¶

    Create /scratch/$USER/sweep-demo/sweep_train.py:

    """Training script that reads hyperparameters from CLI and logs to W&B."""
import argparse
import os

import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset
import wandb


def make_dataset(n_samples, input_dim=16, seed=0):
    gen = torch.Generator().manual_seed(seed)
    X = torch.randn(n_samples, input_dim, generator=gen)
    y = (X[:, :8] * X[:, 8:]).sum(dim=1) + 0.1 * torch.randn(
        n_samples, generator=gen
    )
    return TensorDataset(X, y)


class SensorNet(nn.Module):
    def __init__(self, input_dim=16, hidden_dim=128, dropout=0.0):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_dim, 1),
        )

    def forward(self, x):
        return self.net(x).squeeze(-1)


def train(args):
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

    wandb.init(
        project="aicr-sweep-demo",
        config={
            "learning_rate": args.lr,
            "batch_size": args.batch_size,
            "epochs": args.epochs,
            "hidden_dim": args.hidden_dim,
            "dropout": args.dropout,
            "seed": args.seed,
            "array_task_id": os.environ.get("SLURM_ARRAY_TASK_ID", "none"),
        },
    )

    torch.manual_seed(args.seed)
    train_set = make_dataset(50000, seed=args.seed)
    val_set = make_dataset(10000, seed=args.seed + 1)

    train_loader = DataLoader(
        train_set, batch_size=args.batch_size, shuffle=True, pin_memory=True
    )
    val_loader = DataLoader(
        val_set, batch_size=args.batch_size, pin_memory=True
    )

    model = SensorNet(hidden_dim=args.hidden_dim, dropout=args.dropout).to(device)
    optimizer = torch.optim.Adam(model.parameters(), lr=args.lr)
    loss_fn = nn.MSELoss()

    for epoch in range(1, args.epochs + 1):
        model.train()
        epoch_loss = 0.0
        for X_batch, y_batch in train_loader:
            X_batch, y_batch = X_batch.to(device), y_batch.to(device)
            pred = model(X_batch)
            loss = loss_fn(pred, y_batch)
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            epoch_loss += loss.item() * X_batch.size(0)

        epoch_loss /= len(train_set)

        # Validate
        model.eval()
        val_loss = 0.0
        with torch.no_grad():
            for X_batch, y_batch in val_loader:
                X_batch, y_batch = X_batch.to(device), y_batch.to(device)
                val_loss += loss_fn(model(X_batch), y_batch).item() * X_batch.size(0)
        val_loss /= len(val_set)

        wandb.log({
            "epoch": epoch,
            "train/loss": epoch_loss,
            "val/loss": val_loss,
        })

        print(f"Epoch {epoch:>3d}/{args.epochs}  "
              f"train_loss={epoch_loss:.4f}  val_loss={val_loss:.4f}")

    # Log final metric for sweep comparison
    wandb.log({"final_val_loss": val_loss})

    # Save checkpoint
    output_dir = os.path.expandvars("/scratch/$USER/sweep-demo/results")
    os.makedirs(output_dir, exist_ok=True)
    task_id = os.environ.get("SLURM_ARRAY_TASK_ID", "0")
    ckpt_path = os.path.join(output_dir, f"model_task{task_id}.pt")
    torch.save(model.state_dict(), ckpt_path)
    print(f"Checkpoint saved to {ckpt_path}")

    wandb.finish()


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--lr", type=float, required=True)
    parser.add_argument("--batch-size", type=int, default=256)
    parser.add_argument("--epochs", type=int, default=20)
    parser.add_argument("--hidden-dim", type=int, default=128)
    parser.add_argument("--dropout", type=float, default=0.0)
    parser.add_argument("--seed", type=int, default=42)
    args = parser.parse_args()
    train(args)

    4. Write the Job Array Script¶

    Create sweep_job.sh. The array defines 20 tasks, each using a different learning rate from a predefined list:

    #!/bin/bash
#SBATCH --job-name=hp-sweep
#SBATCH --partition=rtx-batch
#SBATCH --gpus=1
#SBATCH --cpus-per-task=8
#SBATCH --mem=32G
#SBATCH --time=02:00:00
#SBATCH --account=ACCOUNT_NAME
#SBATCH --array=0-19%5
#SBATCH --output=%x-%a-%j.out

module purge

eval "$(conda shell.bash hook)"
conda activate /scratch/$USER/conda-envs/sweep

export WANDB_API_KEY=$(cat ~/.wandb_api_key)
export WANDB_DIR=/scratch/$USER/wandb

mkdir -p "${WANDB_DIR}"

# Define 20 learning rates on a log scale from 1e-5 to ~0.56
LEARNING_RATES=(
    0.00001 0.0000178 0.0000316 0.0000562 0.0001
    0.000178 0.000316 0.000562 0.001 0.00178
    0.00316 0.00562 0.01 0.0178 0.0316
    0.0562 0.1 0.178 0.316 0.562
)

LR=${LEARNING_RATES[$SLURM_ARRAY_TASK_ID]}

echo "Task $SLURM_ARRAY_TASK_ID: learning_rate=$LR"

python /scratch/$USER/sweep-demo/sweep_train.py \
    --lr $LR \
    --batch-size 256 \
    --epochs 20 \
    --seed 42

    Note

    --array=0-19%5 creates 20 tasks but limits concurrency to 5 at a time. This leaves resources available for other users and helps your fairshare score.

    5. Submit the Sweep¶

    $ sbatch sweep_job.sh
Submitted batch job 12345

    Monitor progress:

    $ squeue -u $USER
$ sacct -j 12345 --format=JobID,JobName,State,Elapsed

    6. Analyze Results in W&B¶

    After all tasks complete, open the W&B dashboard at wandb.ai. The project aicr-sweep-demo contains all 20 runs. Use the dashboard to:

    • Sort runs by final_val_loss to find the best learning rate
    • Create a parallel coordinates plot to visualize learning rate vs. validation loss
    • Compare training curves across all 20 runs

    If you used offline mode (or want to sync from a login node), sync W&B logs afterward:

    $ export WANDB_API_KEY=$(cat ~/.wandb_api_key)
$ wandb sync /scratch/$USER/wandb/offline-run-*

    Expected Output¶

    Each task produces a separate output file:

    Task 8: learning_rate=0.001
Epoch   1/20  train_loss=0.8432  val_loss=0.7891
Epoch   2/20  train_loss=0.2134  val_loss=0.1987
...
Epoch  20/20  train_loss=0.0089  val_loss=0.0081
Checkpoint saved to /scratch/USERNAME/sweep-demo/results/model_task8.pt

    In the W&B dashboard, you will see all 20 runs with their learning rates and final validation losses, letting you quickly identify the optimal learning rate.

    Troubleshooting¶

    Problem Cause Fix
    W&B logs not appearing in dashboard API key not set Check WANDB_API_KEY is exported. Compute nodes have outbound internet, so online mode should work.
    Some array tasks fail while others succeed Individual task OOM or code error Check the specific task output file (hp-sweep-TASKID-JOBID.out); resubmit failed tasks with --array=TASKID
    All tasks use the same learning rate $SLURM_ARRAY_TASK_ID not indexing the array Ensure the variable is used as a bash array index (${LEARNING_RATES[$SLURM_ARRAY_TASK_ID]})
    W&B logs filling home quota W&B writes to home by default Set WANDB_DIR=/scratch/$USER/wandb in your job script
    Array job stuck pending Too many concurrent tasks for the partition Reduce the concurrency limit (e.g., %3 instead of %5)
