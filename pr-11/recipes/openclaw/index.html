    OpenClaw AI Assistant¶

    Deploy OpenClaw, an open-source AI assistant, on AICR. OpenClaw connects to cloud LLM providers (Anthropic Claude, OpenAI, Google Gemini, OpenRouter) and runs in an Apptainer container on a compute node. You interact through a web dashboard accessed via SSH tunnel.

    Goal¶

    Set up OpenClaw on AICR so you can use an AI coding assistant that has access to your files and can submit Slurm jobs on your behalf.

    Prerequisites¶

    • An AICR account with a valid Slurm account (--account=ACCOUNT_NAME)
    • An API key from at least one LLM provider (Anthropic, OpenAI, Google, or OpenRouter)
    • SSH access to AICR (see SSH)

    Data Privacy

    OpenClaw sends prompts and file excerpts to cloud LLM APIs over HTTPS. Do not use it with sensitive, confidential, or export-controlled data. You are responsible for the actions of your agent.

    Steps¶

    1. Clone the Repository¶

    $ ssh USERNAME@login.aicr.ai
$ mkdir -p /scratch/$USER/openclaw
$ cd /scratch/$USER/openclaw
$ git clone https://github.com/qsimeon/openclaw-engaging.git openclaw-aicr
$ cd openclaw-aicr

    2. Build and Configure¶

    The setup takes about 15–20 minutes. Run it as an interactive job on the cpu partition:

    $ salloc --partition=cpu --mem=8G --time=01:00:00 --cpus-per-task=2 --account=ACCOUNT_NAME
$ cd /scratch/$USER/openclaw/openclaw-aicr
$ ./apptainer/setup.sh

    The setup wizard will prompt you to:

    • Choose your LLM provider(s)
    • Enter your API key(s)
    • Configure HPC-specific settings

    You can also split this into two steps:

    # Build only (non-interactive)
$ srun --partition=cpu --mem=8G --time=01:00:00 --cpus-per-task=2 --account=ACCOUNT_NAME \
    ./apptainer/setup.sh --build-only

# Configure only (interactive)
$ salloc --partition=cpu --mem=4G --time=00:30:00 --account=ACCOUNT_NAME
$ ./apptainer/setup.sh --onboard-only
$ exit

    3. Activate the Module¶

    Add the OpenClaw module to your environment (one-time setup):

    $ echo 'module use ~/modulefiles' >> ~/.bashrc && source ~/.bashrc
$ module load openclaw
$ openclaw --help

    4. Test the Agent¶

    Send a test message to verify everything works:

    $ openclaw agent --local --agent main -m "Hello from AICR!"

    5. Launch the Web Dashboard¶

    Start the gateway as a Slurm job:

    $ cd /scratch/$USER/openclaw/openclaw-aicr
$ ./apptainer/start-gateway.sh

    Or submit manually:

    $ sbatch apptainer/slurm-gateway.sh
$ cat openclaw-gw-JOBID.out

    The output will show the node name and port. Set up an SSH tunnel from your local machine:

    $ ssh -f -N -L 18790:NODE_NAME:18790 USERNAME@login.aicr.ai

    Then open http://localhost:18790 in your browser.

    Tip

    Replace NODE_NAME with the actual compute node from the job output. Find it with squeue -u $USER.

    Giving the Agent Access to Your Data¶

    By default, OpenClaw runs in --containall mode with restricted filesystem access. To let the agent read your data, bind-mount specific directories:

    $ APPTAINER_BIND="/scratch/$USER/my-project" openclaw agent --local --agent main \
    -m "Explore the CSV files in /scratch/$USER/my-project/"

    For the web dashboard with data access:

    $ APPTAINER_BIND="/scratch/$USER/my-project" ./apptainer/start-gateway.sh

    Submitting Slurm Jobs from the Agent¶

    OpenClaw can submit Slurm jobs on your behalf — for example, launching GPU training runs while the agent itself runs on a CPU node:

    $ OPENCLAW_SLURM_BINDS=1 ./apptainer/start-gateway.sh

    Or in a local session:

    $ OPENCLAW_SLURM_BINDS=1 APPTAINER_BIND="/scratch/$USER/my-project" \
    openclaw agent --local --agent main \
    -m "Write a Slurm batch script to train a model on the B200 partition and submit it"

    Warning

    When OPENCLAW_SLURM_BINDS=1 is set, the agent can submit jobs that run outside the container — on GPU nodes with full filesystem access. Review the agent's actions carefully.

    AICR-Specific Configuration¶

    Partitions for OpenClaw¶

    What Partition Why
    OpenClaw gateway/agent cpu No GPU needed — the agent calls cloud APIs
    GPU jobs submitted by agent rtx-batch, b200-batch Agent writes and submits GPU job scripts
    Interactive testing cpu (with salloc) Quick tests during setup

    Cluster-Aware Agent¶

    Initialize the agent with knowledge of AICR's partitions, storage layout, and documentation:

    $ ./apptainer/orcd-workspace-init.sh

    Changing the LLM Model¶

    $ openclaw config set agent.model "anthropic/claude-sonnet-4-6"

    Updating OpenClaw¶

    $ cd /scratch/$USER/openclaw/openclaw-aicr
$ ./apptainer/update.sh

    Check for updates without applying:

    $ ./apptainer/update.sh --check

    Rebuilding the Container¶

    If you need to rebuild from scratch:

    $ salloc --partition=cpu --mem=8G --time=01:00:00 --cpus-per-task=2 --account=ACCOUNT_NAME
$ module load apptainer
$ apptainer build apptainer/openclaw.sif apptainer/openclaw.def
$ exit

    Environment Variables¶

    Variable Default Purpose
    OPENCLAW_SLURM_BINDS off Enable agent to submit Slurm jobs
    OPENCLAW_CONTAINALL on Filesystem isolation (0 to disable)
    OPENCLAW_GATEWAY_PORT 18790 Gateway web interface port
    APPTAINER_BIND — Additional directories to mount in the container

    Troubleshooting¶

    Problem Solution
    Setup fails with out-of-memory Increase --mem in the salloc/srun command (try 12G)
    Cannot reach dashboard Check the SSH tunnel is running and the node name is correct
    Agent can't see my files Set APPTAINER_BIND="/scratch/$USER/my-dir" before launching
    Token auto-fill not working Find the token manually: python3 -c "import json; print(json.load(open('.openclaw/openclaw.json'))['gateway']['auth']['token'])"
    Agent can't submit Slurm jobs Set OPENCLAW_SLURM_BINDS=1 before launching
    Port already in use Kill old tunnels: lsof -ti:18790 \| xargs kill -9 then reconnect
