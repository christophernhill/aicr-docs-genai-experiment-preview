    Getting Started InfiniBand

    Networking¶

    AICR uses four distinct network fabrics, each purpose-built for a specific traffic class. Two are InfiniBand (high-performance compute and storage) and two are Ethernet (management and hardware control). This separation ensures that GPU training traffic, storage I/O, management operations, and hardware control never compete for the same links.

    Fabric Overview¶

    Fabric Technology Switches Purpose Connected Nodes
    B200 Rail-Optimized IB NDR InfiniBand 12 (2 spine + 10 leaf) GPU-to-GPU training across B200 nodes B200 only
    Cluster IB NDR InfiniBand 6 (2 spine + 4 leaf) Storage (VAST), MPI, all nodes All
    Core Ethernet 25/100/400 GbE 2x SN4700 (MLAG pair) Management, provisioning, user access All
    Out-of-Band 1 GbE 9x SN2201 (1 per rack) BMC/iDRAC hardware management All

    Per-Node Connectivity¶

    Node Type Count Rail-Optimized IB Cluster IB Core Ethernet OOB
    B200 31 8x NDR 400G 1x NDR 400G 2x 25 GbE 1x 1 GbE
    RTX PRO 6000 19 — 2x NDR 400G 2x 25 GbE 1x 1 GbE
    Service 32 — 1x NDR 400G 2x 100 GbE 1x 1 GbE

    1. B200 Rail-Optimized InfiniBand¶

    The highest-performance fabric in the cluster. It carries GPU-to-GPU traffic for distributed training across the 31 B200 nodes exclusively — no other node type connects to it.

    Rail Optimization¶

    Each B200 node has 8 InfiniBand NICs, one per GPU. In the rail-optimized layout, all GPU[0]s across every node connect to the same leaf switch ("Rail 0"), all GPU[1]s to another leaf ("Rail 1"), and so on through Rail 7. This aligns with data-parallel training, where each GPU synchronizes gradients with the corresponding GPU on other nodes — that allreduce traffic stays within a single leaf switch hop.

    Property Value
    Topology Rail-optimized leaf-spine (SuperPOD reference)
    Leaf switches 10x NVIDIA QM9790 (NDR, 800G-capable)
    Spine switches 2x NVIDIA QM9700 (NDR)
    Node-to-leaf speed NDR 400 Gb/s per link
    Spine-leaf speed NDR 800 Gb/s (bonded 2x400G)
    Total node ports 248 (31 nodes x 8 GPUs)
    Bandwidth per node 3,200 Gb/s (400 GB/s aggregate)
    Managed by 2x NVIDIA R660 UFM appliances

    Expansion¶

    The current 31 nodes nearly fill one NVIDIA Scalable Unit (32-node capacity). The spine switches already support a second SU, so the cluster can grow to 63 B200 nodes (504 GPUs) by adding leaf switches and cabling — no recabling of existing nodes required.

    2. Cluster InfiniBand¶

    The cluster-wide general-purpose InfiniBand network connecting every node type — B200, RTX, service, and VAST storage. It carries storage I/O and MPI communication for RTX multi-node jobs.

    Property Value
    Topology Leaf-spine
    Leaf switches 4x NVIDIA QM9790
    Spine switches 2x NVIDIA QM9700
    Total endpoint ports 133
    Aggregate endpoint bandwidth 5,950 GB/s
    Managed by 2x NVIDIA R660 UFM appliances

    Connected Endpoints¶

    Endpoint Ports Speed Aggregate BW
    B200 nodes (31) 31 NDR 400G 1,550 GB/s
    RTX nodes (19) 38 NDR 400G 1,900 GB/s
    Service nodes (32) 32 NDR 400G 1,600 GB/s
    VAST CBoxes (14) 28 NDR 200G 700 GB/s
    UFM appliances (2) 4 NDR 400G 200 GB/s

    RTX nodes have 2 IB ports on this fabric, used flexibly for both storage and MPI. The 100 GB/s aggregate per RTX node is ample for VAST parallel I/O and moderate-scale distributed training. Service nodes each have 1 IB port, ensuring that Globus transfers and file staging run at full InfiniBand speed to VAST.

    3. Core Ethernet¶

    Handles management, provisioning, user SSH sessions, web portals (OnDemand), and IP-based services. Does not carry GPU compute or storage I/O — those run over InfiniBand.

    Property Value
    Switches 2x NVIDIA SN4700 (Spectrum-3, MLAG pair)
    GPU nodes 2x 25 GbE each (lightweight management)
    Service nodes 2x 100 GbE each (gateway role, 4x faster)
    VAST Ethernet CBoxes 400 GbE (IP-based NFS/S3 access)

    Service nodes get 100 GbE because they are the front door — user SSH, OnDemand web traffic, and Globus transfers all flow through these Ethernet connections. GPU nodes only need 25 GbE for management since their heavy I/O runs over InfiniBand.

    4. Out-of-Band Management¶

    Dedicated hardware management access independent of the operating system, connecting to each node's BMC (iDRAC on Dell servers).

    Property Value
    Switches 9x NVIDIA SN2201 (1 per rack)
    Ports per switch 48x 1 GbE + 4x 100 GbE uplink
    Total endpoints 107 (all nodes + VAST devices)
    Purpose iDRAC, remote console, power control, firmware updates

    The OOB network is intentionally separate from all data and management traffic. Even if InfiniBand and Ethernet are completely down, administrators can reach every device via its BMC.

    Fabric Management: NVIDIA UFM¶

    All InfiniBand switches are managed by 4 NVIDIA R660 UFM appliances in redundant pairs:

    UFM Pair Managed Fabric Switches
    2x R660 B200 rail-optimized IB 12 switches
    2x R660 Cluster IB 6 switches

    UFM provides real-time fabric monitoring, topology discovery, adaptive routing, SHARP (in-network allreduce acceleration), cable diagnostics, and telemetry streaming.

    Aggregate Bandwidth¶

    Fabric Total Endpoint BW Primary Traffic
    B200 Rail IB 12,400 GB/s GPU training (allreduce)
    Cluster IB 5,950 GB/s Storage I/O, MPI
    Core Ethernet ~230 Gb/s Management, user access
    OOB ~107 Mb/s Hardware control
